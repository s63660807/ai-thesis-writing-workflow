# 目标指令：论文写作前期准备与 Skill 加载

你现在接手的是一个可复用的论文写作准备包。请把本文件所在文件夹视为项目根目录，先完成论文写作前期准备工作，再根据用户填写的《论文信息采集表.md》进入选题、文献检索、框架搭建和分章写作。

本指令适用于 Codex、Workbuddy 以及其他支持本地文件读取、项目资料导入或自定义技能的智能体。不要默认用户一定使用 Codex。

## 一、你的总目标

1. 从本文件夹的 `skills/` 目录加载、导入、安装或读取论文写作相关 skills。
2. 检查本机是否具备当前智能体、Python、Node.js、Git 等基础运行条件。
3. 读取并遵守 `论文写作流程.md`。
4. 读取 `环境准备说明.md`，理解外部软件只检查和引导安装，不把安装包本体放进分享包。
5. 读取 `论文信息采集表.md`，根据缺失项向用户提出必要问题。
6. 建立论文项目的前期文件结构、文献仓库和参考文献校验机制。
7. 进入论文前期工作：选题澄清、研究方向确认、关键词和检索式生成、文献检索与核验表搭建、章节大纲生成。

## 二、必须遵守的原则

1. 不编造文献、数据、访谈、政策文件、页码或引用结论。
2. 机器检索、PDF 抽取、格式检查只能算机器处理；需要人工核验的地方必须标记为“待人工核验”。
3. 正文引用、参考文献列表、参考文献校验表必须同步维护。
4. 每完成一个关键阶段，输出简短状态报告，说明已完成项、待用户确认项、风险项。
5. 中文文件、CSV、Markdown、PDF 抽取内容优先按 UTF-8 读取和写入。
6. 不要把大段正文、表格、JSON、CSV 全量打印到控制台；长内容先写文件，再输出路径、记录数、字数和少量样例。

## 三、先加载或安装本地 skills

优先使用本文件夹自带的 `skills/` 目录，不要一开始就从互联网重新下载。根据当前智能体软件的能力选择合适方式：

1. 如果当前软件支持“导入技能”“导入知识库”“项目资料”“本地文件夹读取”，请直接导入或读取本文件夹和 `skills/` 目录。
2. 如果当前软件没有正式的 skill 安装机制，请把 `skills/` 作为本地指令库使用：执行某类任务前，先读取对应目录下的 `SKILL.md`。
3. 如果当前软件是 Codex，可以使用下面的 PowerShell 脚本把 skills 同步到 Codex 的本机 skills 目录。

Codex 用户可选执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\install-skills.ps1
```

如果当前软件不是 Codex，不要强行写入 `$HOME\.codex\skills`。请先确认该软件自己的技能、插件或知识库导入方式；如果无法确认，就保持 `skills/` 在当前项目内，按需读取 `SKILL.md`。

如果需要手动安装到某个智能体的 skills 目录，请完成同等动作：

1. 找到本文件夹下的 `skills/`。
2. 找到当前软件官方认可的 skills、插件、知识库或项目资料目录。
3. 将 `skills/` 下每个包含 `SKILL.md` 的子目录复制或导入到目标目录。
4. 已存在同名 skill 时，以合并更新为主，不要删除用户原有自定义文件，除非用户明确要求覆盖清理。

加载或安装后请列出已识别的 skill 名称，并确认至少包含：

- `research-writing-skill`
- `pdf`
- `gs-search`
- `gs-advanced-search`
- `gs-fulltext`
- `cnki-search`
- `cnki-advanced-search`
- `cnki-paper-detail`

## 四、检查运行环境

请检查以下环境，并把结果写入 `logs/environment-check.md`：

1. 当前工作目录。
2. Python 是否可用。
3. Node.js/npm/npx 是否可用。
4. Git 是否可用。
5. 当前智能体是否能读取本地文件。
6. 本地 skills 是否已加载、导入、安装，或是否能作为本地指令库被读取。
7. 如需使用 CNKI MCP、CNKI 浏览器自动化、Google Scholar、https://xs.xasa.top/ 等能力，检查浏览器控制、登录态、下载权限和验证码状态是否可用。

如果当前系统可以运行 PowerShell，优先执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\check-environment.ps1
```

如果检查发现缺少基础软件，先向用户说明缺失项。Windows 用户可以选择运行 `install-dependencies-winget.ps1`，但不得在未告知用户的情况下静默安装系统软件。

注意：CNKI、Google Scholar、https://xs.xasa.top/、Zotero、付费数据库、学校图书馆入口、验证码、登录态和下载权限可能需要用户手动配合。不要把“skill 已加载或已安装”说成“数据库检索已完全可用”。

## 五、建立论文项目基础结构

在项目根目录创建或确认以下目录和文件：

```text
References_PDF/
chapters/
refs/
scripts/
outputs/
logs/
```

在 `refs/` 下创建或确认：

```text
refs/reference-verification.csv
refs/literature-search-plan.md
refs/evidence-map.md
```

`reference-verification.csv` 至少包含这些字段：

```csv
引用来源,论文引用位置,是否人工校验,校验状态,校验情况,用户校验结果,处理建议,最后更新时间
```

在 `chapters/` 下创建或确认：

```text
chapters/chapter-snapshot.md
chapters/outline.md
```

## 六、读取用户信息并启动前期写作

请按以下顺序执行：

1. 读取 `论文信息采集表.md`，提取论文类型、专业、目标字数、文献要求、禁止事项、导师偏好等。
2. 读取 `论文写作流程.md`，以其中的流程作为项目主规范。
3. 若信息采集表存在“不清楚”“不确定”“无”等关键缺口，先用少量问题向用户澄清。
4. 如果用户还没有明确题目，先进行选题头脑风暴，输出 3-5 个可行选题，并说明研究对象、问题意识、资料可得性和风险。
5. 如果用户已有明确题目，直接生成：
   - 研究问题初稿
   - 关键词表
   - 中文检索式
   - 英文检索式
   - 文献检索计划
   - 论文一级/二级目录草案
6. 对可检索文献，优先使用已安装的 Google Scholar、CNKI、PDF 相关 skills；英文文献可补充使用学术搜索网站 `https://xs.xasa.top/`。如果当前智能体支持脚本、MCP 或浏览器自动化，可用脚本/MCP查询关键词、题名、作者、DOI等信息，并记录检索式、命中文献、PDF/DOI链接和获取状态；遇到权限、验证码或无法下载全文时，明确交给用户兜底。
7. 中文文献优先使用 CNKI MCP/浏览器自动化批量检索、排序、下载和读取。筛选时优先考虑被引量高、下载量高、来源权威性高的论文，重点关注 SCI、SSCI、CSCD、CSSCI、北大核心、EI、中科院分区表、JCR分区表等主流收录/分区信号；不能在数据库页面或用户材料中核验的来源级别必须标记为“待核验”。

## 七、文献与引用工作要求

1. 所有 PDF 统一放入 `References_PDF/`。
2. 新增文献必须同步记录到 `refs/reference-verification.csv`。
3. 只有完成元数据核验和内容核验后，才能标记为“已通过”。
4. 不能确认的信息保持“待校验”，不要补猜。
5. 正文中每插入一条引用，都要同步更新参考文献章节和校验表。
6. 尽量避免连续堆叠引用，优先让每条引用自然支撑具体句子。

## 八、阶段性输出

前期准备完成后，请给用户一个简短报告，至少包括：

1. skills 加载或安装状态。
2. 环境检查结果。
3. 已创建的目录和文件。
4. 从信息采集表提取到的论文约束。
5. 需要用户补充或人工确认的问题。
6. 下一步建议：选题确认、检索计划确认或直接进入开题/大纲。

如果任何步骤失败，请说明失败原因、已经完成的部分、可手动补救的方法。
