# 环境准备说明

本分享包的核心是 `skills/` 和论文写作流程文件，不建议把 Python、Node.js、Git、Pandoc、Chrome 等安装包本体放进文件夹。

推荐做法是：先运行环境检查脚本，缺什么再按官方渠道或系统包管理器安装什么。

## 一、需要准备的软件

建议优先检查这些基础软件：

| 软件 | 用途 | 是否必须 |
| :--- | :--- | :--- |
| Python | 读取/处理 PDF、CSV、Excel、Markdown，运行批处理脚本 | 建议必备 |
| Node.js / npm / npx | 支持部分智能体工具、浏览器自动化、MCP 或前端工具链 | 建议必备 |
| Git | 下载公开仓库、同步资料、版本管理 | 建议必备 |
| Pandoc | 将 Markdown 论文转换为 Word/docx | 写终稿时需要 |
| Chrome 或 Edge | CNKI、Google Scholar、Zotero、网页检索和浏览器自动化 | 检索时常用 |
| 当前智能体软件 | Codex、Workbuddy 或其他支持本地文件读取的 AI 工具 | 必备 |

## 二、先检查环境

在本文件夹根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\check-environment.ps1
```

脚本会检查常见命令是否可用，并生成：

```text
logs/environment-check.md
```

这个脚本只检查，不安装，不修改系统环境。

## 三、Windows 用户可选自动安装

如果电脑支持 `winget`，可以运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\install-dependencies-winget.ps1
```

该脚本会尝试安装：

- Python 3
- Node.js LTS
- Git
- Pandoc
- Google Chrome

注意：

1. `winget` 可能需要用户确认安装协议。
2. 安装后可能需要重新打开 PowerShell 或重启智能体软件，PATH 才会刷新。
3. 如果用户使用 Workbuddy 或其他软件，并且软件自带 Python/Node 环境，也可以不安装重复组件。

## 四、不要把安装包本体放进分享包

不建议把 `.exe`、`.msi` 安装文件直接放进本文件夹，原因是：

1. 安装包体积大，分享成本高。
2. 版本容易过期。
3. 用户更信任官方渠道。
4. 不同用户缺的软件不同，不适合统一打包。

这个分享包应保持轻量：保留流程、目标指令、采集表、skills、检查脚本和安装指引即可。
